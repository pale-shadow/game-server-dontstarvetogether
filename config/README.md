# Don't Starve Together Config Files

This README describes the files in this saves directory and their purpose for running a Don't Starve Together dedicated server.

## Files

- **modoverrides.lua**
  - Purpose: Active mod configuration for the world. Lists enabled mods and any per-mod configuration (enabled/disabled flags, mod-specific settings) that the server will load for the world.
  - When to edit: Change to enable/disable mods for the server world or to tweak mod settings that need to persist across server runs.

- **modoverrides.lua.backup**
  - Purpose: Backup copy of a previous or known-good `modoverrides.lua`.
  - When to use: Restore if the active `modoverrides.lua` becomes corrupted or you need to roll back to a prior mod configuration.

- **mod_list.csv**
  - Purpose: Simple list of installed/available mods with their IDs and metadata (used by some tools or scripts to determine available mods).
  - When to edit: Updated when adding or removing mods manually; typically generated by mod management scripts.

- **README.md**
  - Purpose: Short placeholder README present before this file was generated.
  - When to edit: Replace or expand with project-specific notes, instructions, or change history.

- **worldgenoverride.lua**
  - Purpose: World generation overrides applied when creating a world (biome size/density, resource distribution, season settings, etc.). Controls how the world is generated for new or reset worlds.
  - When to edit: Tune world generation parameters before creating a new server world. Existing worlds are not affected unless regenerated.

- **worldgenoverride.lua-caves**
  - Purpose: World generation overrides specific to cave (Cave/DS) world generation. Controls cave-specific features, resource placement, and special cave mechanics.
  - When to edit: Used when enabling or customizing cave worlds for your server.

- **gen_mod_override.sh**
  - Purpose: Shell script to generate or update `modoverrides.lua` or other mod configuration artifacts automatically (commonly used to build `modoverrides.lua` from a template or a mod list).
  - When to run: After changing `mod_list.csv` or when automating mod configuration deployment.

- **announcemod.txt**
  - Purpose: Plain-text announcement or message used by a mod (commonly a server announcement mod) to broadcast server rules, events, or notices to players when they join.
  - When to edit: Update server announcements, rules, or notices to be shown to players on join.

- **dedicated_server_mods_setup.lua**
  - Purpose: Server-side Lua script used for setting up mods at the dedicated server level. Often used to ensure required mods are loaded, to perform server-only mod setup tasks, or to register mod-related behavior on startup.
  - When to edit: Modify for advanced server mod initialization or to add server-only mod logic.

- **cluster.ini**
  - Purpose: Cluster configuration file for the server. Defines cluster-level settings such as cluster name, master/slave ports, cluster token, and per-server (shard) settings. Crucial for multi-shard setups and server discovery.
  - When to edit: Update when reconfiguring cluster settings, ports, or adding/removing shards.

## Directory Notes

- Parent directory entries (..) indicate an encompassing folder (likely the saves root that contains multiple worlds/clusters).
- Permissions on scripts (e.g., `gen_mod_override.sh` is executable) indicate they are intended to be run on the host to automate configuration tasks.

## Typical Workflows

1. Add or remove mods:
   - Update `mod_list.csv` (or use workshop/launcher).
   - Run `gen_mod_override.sh` to regenerate `modoverrides.lua`.
   - Restart server.

2. Change world generation:
   - Edit `worldgenoverride.lua` and/or `worldgenoverride.lua-caves`.
   - Generate a new world or recreate existing world if desired.

3. Update announcements:
   - Edit `announcemod.txt`.
   - Restart or reload the announcement mod if required.

4. Restore configuration:
   - Replace `modoverrides.lua` with `modoverrides.lua.backup` if needed.

## Backup Recommendations

- Keep regular backups of `modoverrides.lua`, `worldgenoverride.lua`, `cluster.ini`, and `dedicated_server_mods_setup.lua`.
- Version control (git) is useful for tracking changes to configuration files and scripts.
